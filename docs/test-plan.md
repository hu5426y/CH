# 测试方案与用例

## 1. 测试目标

验证系统满足文档中的功能性与非功能性要求：
- 功能完整性
- 权限与安全
- 并发与性能
- 兼容性

## 2. 功能测试建议

- F-001 登录：正确账号密码可登录，错误密码拒绝。
- F-002 注册：新学生可注册，重复账号拒绝。
- F-003 座位预约：空闲座位可预约，状态转为 RESERVED。
- F-004 重复预约拦截：同一学生存在有效预约时禁止再次预约。
- F-005 冲突预约拦截：同一时间段同一座位只能成功一单。
- F-006 取消预约：预约开始前可取消，座位回到 FREE。
- F-007 签到：在签到窗口内可签到，状态变 CHECKED_IN。
- F-008 签离：签到后可签离，状态变 COMPLETED。
- F-009 超时未签到：定时任务将预约置为 VIOLATED，并记录违约。
- F-010 违约撤销：管理员可撤销 ACTIVE 违约并恢复积分。
- F-011 规则配置：修改规则后新预约流程生效。
- F-012 操作日志：关键操作后可在日志列表查到记录。

## 3. 安全测试建议

- S-001 未登录访问受保护接口应返回 401。
- S-002 学生访问管理员接口应返回 403。
- S-003 密码库表存储应为 BCrypt 哈希。
- S-004 高频请求触发 429 限流。

## 4. 性能测试建议

JMeter 场景：
- 200 并发同时查询座位
- 200 并发同时发起预约
- 100 查询 + 100 预约混合场景

目标：平均响应时间 < 2s，错误率接近 0。

## 5. 兼容性建议

- 浏览器：Chrome / Edge / Firefox
- 分辨率：1920x1080、1366x768、移动端竖屏
